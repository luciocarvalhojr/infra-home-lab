# infra-home-lab

Home lab infrastructure managed with Terraform + Ansible on Proxmox VE.

## current state

- **Hypervisor:** Proxmox VE (single node: `hyper01` at 192.168.0.90)
- **Kubernetes:** K3s v1.34.5+k3s1 — 1 control plane + 3 workers (192.168.0.130–133)
- **Object storage:** Minio LXC on Proxmox (Alpine 3.21) at 192.168.0.95 — used as Terraform S3 remote backend (bucket: `terraform-state`)
- **DNS:** Technitium integrated via `kenske/technitium` provider; zone `home.lan`
- **Terraform module:** `github.com/luciocarvalhojr/terraform-modules//k3s-node` @ v1.2.1
- **Pipeline:** GitHub Actions runs on push/PR to `main` — jobs: secret scan (gitleaks), terraform fmt/validate/lint/checkov, ansible-lint
- **Deployed:** Yes — cluster is live; Terraform remote state is active on Minio

## in progress

## key decisions

- **Minio for Terraform state (not cloud):** keeps state local; S3-compatible API allows future migration to AWS; runs independently of the K3s cluster so state remains accessible when cluster is down
- **External Terraform module (`k3s-node`):** encapsulates VM + cloud-init + DNS record creation; versioned independently; controlplane and workers share the same module with different parameters
- **Ansible inventory generated by Terraform:** `inventory.tpl` Jinja2 template produces `ansible/inventory.yml` as a `local_file` resource — Terraform is the single source of truth for node IPs/names
- **`--cluster-init` on control plane:** enables embedded etcd instead of SQLite, making the cluster HA-ready without a topology change
- **`serial: 1` in upgrade playbook:** rolling worker upgrades one at a time to preserve availability
- **Pre-commit hooks mirror CI:** same checks (gitleaks, tflint, terraform-docs, checkov, ansible-lint) run locally before push
